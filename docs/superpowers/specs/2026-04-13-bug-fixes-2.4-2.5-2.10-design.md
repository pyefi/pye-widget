## 2.4 ‚Äî Position disappears after partial sell

**Problem:** After a sell completes, the user returns to `select-position` and their stake account is missing. It reappears on page refresh. Root cause: `BalanceSyncer` polls every 5 minutes ‚Äî the stale cached data is shown until the next interval fires.

**Fix:**

1. `packages/sdk/src/stores/balance-store.ts` ‚Äî add `stakeRefreshKey: number` to `BalanceState`, `incrementStakeRefreshKey()` to `BalanceActions`, and the corresponding store implementation.
2. `packages/sdk/src/react/BalanceSyncer.tsx` ‚Äî add `stakeRefreshKey` (from `useBalanceStore`) to the `useEffect` dependency array so the effect re-runs (and calls `sync()`) when the key increments.
3. `packages/widget/src/components/screens/ReviewQuote.tsx` ‚Äî after `navigate("complete")` in `handleSign`, call `incrementStakeRefreshKey()` via `useBalanceStore`.

**Constraints:** No change to existing polling behaviour. SDK changes are purely additive.

## 2.5 ‚Äî RedeemComplete broken in production

**Problem:** The `RedeemComplete` screen is scaffolded but unreachable in production:
- `redeemAmountSol` and `redeemTxSignature` have no setter actions in the store
- `handleRedeem` in `HomeScreen` doesn't navigate to `"redeem-complete"` on success ‚Äî only a dev-only button does
- The screen always shows a hardcoded `12.5 SOL` fallback

**Fix:**

1. `packages/widget/src/stores/widget-store.ts` ‚Äî add `setRedeemAmountSol(amount: number)` and `setRedeemTxSignature(sig: string)` to `WidgetActions` and implement them in the store.
2. `packages/widget/src/components/screens/HomeScreen.tsx`:
   - Update `handleRedeem` to capture `{ signature }` from `executeRedeem`
   - Call `setRedeemAmountSol(p.ptAmountLamports / 1e9)` and `setRedeemTxSignature(signature)` on success
   - Call `navigate("redeem-complete")` on success
   - Remove the dev-only preview button block
3. `RedeemComplete.tsx` ‚Äî no changes needed.

## 2.10 ‚Äî Projected earnings identical for Q2‚ÄìQ4

**Problem:** When live order book data is absent, the fallback `parsedAmount * (MARKET_RATE / 100)` is the same constant for all quarters, making all durations show identical projected earnings.

**Fix:**

`packages/widget/src/components/screens/ChooseDuration.tsx` ‚Äî scale the fallback by time-to-maturity:

```ts
const maturityTs = Number(maturities[matId].maturity_timestamp);
const yearsRemaining = Math.max(0, (maturityTs - Date.now() / 1000) / (365.25 * 86400));
const grossYield = bestBid != null
  ? bestBid * parsedAmount
  : parsedAmount * (MARKET_RATE / 100) * yearsRemaining;
```

Live bid data paths are untouched. `MARKET_RATE` is treated as an annual rate; the fallback scales linearly with time remaining to maturity.
